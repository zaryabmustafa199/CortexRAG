# Step 14 — LLM Integration — Query Rewriting & Streaming Response

## What You're Building
The core RAG query ask endpoint (/query/ask). It resolves pronoun-dependent conversational follow-ups into standalone queries based on message history, embeds the query, retrieves and ranks document contexts, and streams the answer using Server-Sent Events (SSE). It handles mid-stream client disconnects gracefully and writes the completed assistant response to the database asynchronously upon stream termination.

---

## Concepts Covered

| Concept | Definition | Why It Matters Here |
|---|---|---|
| **Query Rewriting** | Utilizing the LLM to rewrite follow-up questions containing pronouns into standalone query terms | Ensures multi-turn questions like "what are the trends?" followed by "summarize them" retrieve the correct context |
| **Server-Sent Events (SSE)** | Unidirectional real-time streaming protocol sending event chunks over standard HTTP | Allows tokens to render instantly in the browser as they are generated by the LLM (crucial for UX) |
| **Post-Stream DB Persistence** | Writing message replies to the SQL database only after the generator completes successfully | Keeps conversational history accurate and matches API requirements for asynchronous payloads |
| **Graceful Disconnection** | Clean close handlers that free HTTP connections if the user closes their browser mid-stream | Prevents thread-pooling leaks or hung worker processes from exhausting system resources |

## Engineering Standards Applied (§5)

- **SSE Streaming** — Tokens are yielded immediately using FastAPI's `StreamingResponse` using standard `text/event-stream` media types.
- **Fail-Open Fallback** — Query rewriting errors log warnings and fall back to the raw, unmodified user question, keeping searches functional.
- **Asynchronous Completion** — Database sessions are opened and committed independently inside the generator after the stream completes.
- **RLS isolation** — Sets local session workspace context parameters in the background thread when writing assistant messages.

## Common Errors & Fixes

| Error | Root Cause | Fix |
|---|---|---|
| `ollama_stream_timeout` | Local Ollama inference took longer than 120 seconds | Allocate more CPU/GPU resources to the Ollama container |
| `401 Unauthorized` | Invalid JWT session credentials | Check token headers or re-execute credentials generation |
| Assistant replies are not saved | An exception occurred inside the SSE generator after connection closure | Check the backend container logs to verify database transactions committed successfully |
